## Maps Saga (Evening: 11 May)

### What happened
- User rejected v17-v21 (old pipeline), then test v1-v9 over multiple rounds
- Maps called "terrible" / "rubbish" repeatedly
- Iterated through approaches:
  - v3: matplotlib Natural Earth coastlines (user: "very basic, don't relate to content")
  - v5-v6: Mapbox dark-v11/light-v11 with GeoJSON (user: "no change in this version!")
  - v7-v8: Mapbox hybrid + PIL overlays with Economist style (user: "no change!")
  - v9: Pure PIL with cream bg, no Mapbox dependency — lat/lon swap bug found, markers rendered off-screen at negative coords

### Root cause of map failures
- **Projection math was wrong** for at least 3 versions. `cfg["center"]` stored as `(lat, lon)` but projection code treated it as `(lon, lat)`, putting all markers off-screen
- Static GeoJSON data (same cities/routes every day) didn't adapt to daily content
- Mapbox Static API had no typography control (GeoJSON labels rendered at fixed small sizes)
- User wanted Economist/ISW-level quality — not achievable with available tools in the timeframes given
- Lat/lon projection bug alone made 5+ iterations non-functional without being visible to user (markers off-screen = blank maps)

### Resolution
- **Maps removed from GSIB entirely.** Text callouts replace map sections.
- Cron pipeline updated: `daily-brief-cron.sh` step 2b (maps) removed, step 3 render uses `--no-maps` flag
- `render_brief_magazine.py` updated to handle missing maps gracefully with text callout
